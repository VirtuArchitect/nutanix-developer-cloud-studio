# v2.23.0-execution-broker-hardening

Release date: 2026-07-07

## Summary

This release adds execution broker queue hardening. Broker records create an idempotent, evidence-linked, operator-review-only intake boundary for future controlled adapter execution while real adapter execution remains disabled.

## Added

- Execution broker queue records linked to controlled lab readiness attestations.
- API endpoints for listing and recording broker queue records.
- Admin Operations execution broker panel.
- Idempotency key, approval evidence, kill-switch, and operator-review-only checks.
- Tests proving missing readiness attestations, duplicate idempotency keys, or incomplete evidence block broker readiness.

## Security Notes

- Broker records are queued for operator review only.
- `provisioningEnabled` remains `false`.
- Provider kill-switch state remains disabled.
- Broker queue APIs require the `Platform Admin` role.
- Duplicate idempotency keys block readiness rather than dispatching work.
