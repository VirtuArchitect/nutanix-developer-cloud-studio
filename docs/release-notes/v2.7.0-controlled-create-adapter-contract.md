# v2.7.0-controlled-create-adapter-contract

## Summary

This release defines the disabled AHV create adapter contract for the future controlled VM create path. It maps approved VM sandbox dry-run fields into a Prism Central create payload preview while keeping execute, poll, rollback, and all real mutation operations disabled.

## Added

- Disabled AHV create adapter contract with validate, map payload, execute, poll, and rollback methods.
- API endpoints for listing and recording AHV create adapter contract reviews.
- Admin Control Plane payload preview for future Prism Central create request fields.
- Blocked operation and kill switch evidence in the Admin UI.
- Unit, API, client, and smoke coverage for the contract review path.

## Verification

- `npm.cmd run test -- server/ahvCreateAdapterContract.test.ts server/apiServer.test.ts src/services/cloudStudioApi.test.ts`
- `npm.cmd run build:all`
- Phase gate must pass before promotion.

## Security Notes

- Real AHV create, poll, rollback, power, and delete operations remain disabled.
- The contract maps approved dry-run fields only.
- A future live adapter still requires authorized lab scope, pentest evidence, rollback/destroy proof, and explicit adapter release.
