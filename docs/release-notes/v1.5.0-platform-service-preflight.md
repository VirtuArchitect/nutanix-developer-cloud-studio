# v1.5.0-platform-service-preflight

`v1.5.0-platform-service-preflight` adds fail-closed preflight boundaries for the platform services that follow the VM lifecycle path.

## What Changed

- Added platform-service preflight run records.
- Added a disabled real-adapter preflight interface for NKP, NDB, NUS, and NAI flows.
- Added API endpoints for creating and listing service preflight runs.
- Added Admin Control Plane UI for running service preflight checks.
- Added checks for request validation, VM lifecycle proof, provider readiness, adapter configuration, and real-adapter switch state.
- Added provider-specific blocked operation evidence.

## Safety Boundary

This release does not call NKP, NDB, NUS, or NAI APIs. All service mutation operations are recorded as blocked and `provisioningEnabled` remains `false`.
