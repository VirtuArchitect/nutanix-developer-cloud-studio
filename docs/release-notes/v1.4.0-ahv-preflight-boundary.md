# v1.4.0-ahv-preflight-boundary

`v1.4.0-ahv-preflight-boundary` adds the fail-closed AHV execution boundary needed before any future controlled create adapter work.

## What Changed

- Added AHV controlled-provisioning run records.
- Added a disabled real-adapter preflight interface.
- Added API endpoints for creating and listing AHV preflight runs.
- Added Admin Control Plane UI for running AHV preflight checks.
- Added checks for controlled gate approval, active lab scope, verified lifecycle proof, controlled create switch, and AHV adapter enablement.
- Added audit evidence and automated coverage for preflight behavior.

## Safety Boundary

This release does not call Prism Central. The real adapter is represented as disabled and all AHV mutation operations are recorded as blocked. `provisioningEnabled` remains `false`.
