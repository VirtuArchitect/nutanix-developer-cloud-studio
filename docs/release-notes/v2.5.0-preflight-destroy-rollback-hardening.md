# v2.5.0-preflight-destroy-rollback-hardening

`v2.5.0-preflight-destroy-rollback-hardening` adds explicit rollback and destroy proof before a controlled VM create path can be promoted.

## Added

- Rollback/destroy proof records tied to VM sandbox dry-run plans.
- `/api/vm-sandbox/rollback-destroy-proofs` endpoints for listing and recording proof evidence.
- Controlled provisioning gate checks now require ready rollback/destroy proof.
- Admin Control Plane panel for backup/export evidence, owner notification, teardown order, inventory reconciliation, audit export readiness, and stop conditions.
- Tests proving missing rollback/destroy proof blocks controlled create promotion.

## Security Notes

This release remains evidence-only. It does not call Prism Central, create VMs, power resources on or off, or delete infrastructure. Evidence references must not contain access material.
