# v0.6.0-provisioning-adapters

This release moves Nutanix Developer Cloud Studio from a generic mock control plane toward a viable hosted/on-prem platform architecture.

## Highlights

- Added a provisioning adapter contract for validate, plan, provision, poll, and destroy behavior.
- Added API-backed readiness records for NCI, NKP, NDB, NUS, NCM, and NAI adapter placeholders.
- Added an image/profile catalog covering AHV images, Kubernetes versions, database engines, storage classes, and AI endpoint profiles.
- Added platform configuration references for project, cluster, network, and credential profile planning.
- Added simulated environment destroy lifecycle with teardown control-plane jobs and audit evidence.
- Added admin console panels for Image and Template Catalog and Provider Readiness.
- Added tests and smoke coverage for provider inventory and destroy workflow.

## Safety Boundary

All adapters remain mock-only. The release does not call Nutanix APIs, does not store secrets, does not provision infrastructure, and does not delete infrastructure.
