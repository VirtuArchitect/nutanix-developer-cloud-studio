# v2.86.0 - API Auth Boundary Hardening

## Added

- API auth boundary diagnostics at `/api/auth/boundary-diagnostics`.
- Malformed trusted identity header rejection when `NDC_REQUIRE_TRUSTED_IDENTITY=true`.
- Admin Overview auth boundary evidence for trusted mode, issuer, role claims, and audit recording.

- Health and readiness probes remain public for load balancers.
- API routes fail closed when trusted identity mode is required.
- Real provisioning and real Prism calls remain disabled.
