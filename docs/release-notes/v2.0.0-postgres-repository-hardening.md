# v2.0.0-postgres-repository-hardening

Release date: 2026-07-06

## Summary

This release hardens the production repository boundary without adding a runtime database dependency. Postgres mode now validates configuration before startup, migration files are checked by the phase gate, and the scaffold remains fail-closed until an approved database driver is introduced.

## Added

- Postgres repository configuration validator.
- Startup fail-closed behavior for invalid `NDC_REPOSITORY=postgres` settings.
- Postgres readiness metadata for operator diagnostics.
- Migration scaffold validation script.
- Phase gate coverage for Postgres scaffold validation.
- Unit tests for configuration, schema validation, readiness, and fail-closed behavior.

## Guardrails

- No PostgreSQL runtime dependency was added.
- Connection string values are not logged by validators.
- Database mode remains scaffold-only.
- Real Nutanix mutation adapters remain disabled.
