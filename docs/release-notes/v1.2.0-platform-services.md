# v1.2.0-platform-services

`v1.2.0-platform-services` adds gated platform-service planning flows for the services that would follow a proven VM lifecycle.

## What Changed

- Added platform-service request records for NKP namespace, NDB PostgreSQL, NUS storage, and NAI endpoint flows.
- Added API endpoints for creating and listing platform-service request plans.
- Added validation for published profiles, provider mapping, service naming, environment mapping, and VM lifecycle proof.
- Added Admin Control Plane UI for planning all four service flows.
- Added cost estimates, approval evidence, rollback notes, cleanup plans, and audit events.
- Added automated coverage for API, client, and browser smoke behavior.

## Safety Boundary

This release does not call NKP, NDB, NUS, or NAI APIs. Platform-service requests are planning records only and remain blocked until controlled VM create, verify, rollback, and destroy proof exists. `provisioningEnabled` remains `false`.
