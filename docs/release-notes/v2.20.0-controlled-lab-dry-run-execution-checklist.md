# v2.20.0-controlled-lab-dry-run-execution-checklist

Release date: 2026-07-07

This release adds evidence-only controlled lab dry-run execution checklists. Checklists record final operator readiness evidence before any future live-lab adapter operation can be considered.

## Added

- Controlled lab dry-run execution checklist records linked to rehearsal packets.
- API endpoints for listing and recording dry-run execution checklists.
- Admin Operations dry-run checklist panel.
- Operator roster, observation window, log capture, rollback timer, stop authority, and disabled execution checks.
- Tests proving missing rehearsal packets or checklist evidence block readiness.

## Verification

- `npm.cmd run test -- server/controlledLabDryRunExecutionChecklist.test.ts server/apiServer.test.ts src/services/cloudStudioApi.test.ts`
- `npm.cmd run build:all`
- Full phase gate: `.\scripts\run-phase-gate.ps1 -TargetPhase v2.20.0-controlled-lab-dry-run-execution-checklist`

## Security Notes

- Dry-run execution checklists are evidence-only.
- `provisioningEnabled=false` remains enforced.
- Real Nutanix adapter execution remains disabled.
- Checklist records store readiness references only, not provider data, credentials, tokens, or secrets.
