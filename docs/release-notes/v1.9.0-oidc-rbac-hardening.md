# v1.9.0-oidc-rbac-hardening

Release date: 2026-07-06

## Summary

This release hardens identity and authorization readiness before any live adapter path is considered. It adds optional strict trusted-header mode, session diagnostics, and an operator-visible role/action matrix.

## Added

- `NDC_REQUIRE_TRUSTED_IDENTITY=true` fail-closed mode for API routes.
- Public health/readiness probe exemption for strict identity deployments.
- `/api/session/diagnostics` endpoint.
- Admin Overview identity boundary and authorization matrix panel.
- Server, client, and smoke coverage for diagnostics and RBAC hardening.

## Guardrails

- Trusted identity headers must be produced by a validating reverse proxy or ingress.
- Diagnostics expose roles and boundaries, not sensitive identity claims.
- Real Nutanix mutation adapters remain disabled.
- `provisioningEnabled` remains `false`.
