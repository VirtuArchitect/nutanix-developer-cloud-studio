# v2.8.0-platform-service-adapter-contracts

## Summary

This release defines disabled platform-service adapter contracts for NKP, NDB, NUS, and NAI. Service requests and preflight records can now be reviewed as future provider payload previews while execute, poll, rollback, and all real mutation operations remain disabled.

## Added

- Disabled platform-service adapter contract with validate, map payload, execute, poll, and rollback methods.
- API endpoints for listing and recording service adapter contract reviews.
- Admin Control Plane payload preview for NKP, NDB, NUS, and NAI service requests.
- Provider-specific blocked operation and kill switch evidence.
- Unit, API, client, and smoke coverage for the service contract review path.

## Verification

- `npm.cmd run test -- server/platformServiceAdapterContract.test.ts server/apiServer.test.ts src/services/cloudStudioApi.test.ts`
- `npm.cmd run build:all`
- Phase gate must pass before promotion.

## Security Notes

- Real NKP, NDB, NUS, and NAI execute, poll, rollback, create, update, and delete operations remain disabled.
- Contract payloads map approved request fields only.
- Future live service adapters still require authorized scope, VM lifecycle proof, service preflight, audit evidence, rollback ownership, and explicit adapter release.
