# v2.1.0-audit-export-retention-hardening

Release date: 2026-07-06

## Summary

This release hardens audit export and retention evidence before real provider operations are recorded. Exports now include manifest metadata and a checksum, while Admin Operations shows retention diagnostics and destination status.

## Added

- Audit export manifest records.
- SHA-256 checksum over the retained audit window.
- `/api/audit/retention` diagnostics endpoint.
- Admin Operations retention diagnostics and manifest evidence.
- Audit export destination-reference validator.
- Phase gate coverage for audit export configuration validation.

## Guardrails

- Destination references must not include embedded auth material.
- Export records remain metadata until external storage is configured.
- Real Nutanix mutation adapters remain disabled.
