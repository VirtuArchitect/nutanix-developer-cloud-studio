# v0.9.0-production-foundation

`v0.9.0-production-foundation` adds starter production controls around the hosted/on-prem API while keeping all real provisioning disabled.

## Highlights

- Added OIDC-shaped request context from trusted prototype headers.
- Added RBAC checks for mutating admin, approval, registry, integration, control-plane, destroy, and inventory import routes.
- Added security headers to API and static responses.
- Added in-memory API rate limiting.
- Added structured JSON request logging and request IDs.
- Added JSON request body size guardrail.
- Added audit retention for memory and JSON-file state stores.
- Added Postgres repository scaffold and initial SQL migration.
- Added GitHub Security workflow with CodeQL and dependency review.

## Important Notes

Trusted identity headers are a starter boundary only. Production deployment still requires real OIDC token validation at the API or trusted ingress. The Postgres repository is scaffolded without a runtime database driver because adding a database dependency requires explicit approval.
