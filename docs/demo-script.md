# Nutanix Developer Cloud Studio - Demo Script

## Audience

Stakeholders evaluating whether a Nutanix-powered internal developer platform is useful enough to take into a real integration phase.

## Five Minute Walkthrough

1. Open the developer portal dashboard.
   - Point out active environments, policy pass rate, estimated monthly spend, and recommended golden paths.

2. Open the golden-path catalog.
   - Show that developers choose from approved app patterns instead of assembling infrastructure by hand.

3. Open a template details page.
   - Explain the outcomes, target services, baseline cost, and readiness notes.

4. Launch a new environment.
   - Use a realistic name such as `checkout-api-dev`.
   - Select target services such as Kubernetes, database, and storage.
   - Show the policy, cost, compliance, and integration checks.

5. Watch the environment status page.
   - Show the timed provisioning timeline.
   - For an AI endpoint request, call out the approval pause as a governance pattern.

6. Open the platform admin view.
   - Show integration health for NCI, NKP, NDB, NUS, NCM, and NAI.
   - Edit template owner or tier to demonstrate governance controls.
   - Review integration readiness notes as the bridge toward real Nutanix APIs.

## Closing Message

This MVP proves the workflow and stakeholder value before connecting to production infrastructure. The next investment decision is which real integration should be wired first: Prism Central/NCI, NCM Self-Service, NKP, NDB, NUS, or NAI.
