# SECURITY.md

## Security Policy

Report security vulnerabilities privately. Do not open public issues for
suspected vulnerabilities.

## Supported Versions

Document supported release branches or versions here.

| Version | Supported |
| ------- | --------- |
| main    | Yes       |

## Reporting a Vulnerability

Include:

- Affected component or endpoint.
- Reproduction steps.
- Impact.
- Evidence, redacted where needed.
- Suggested remediation if known.

## Security Review

Security-sensitive code changes should follow `SECURITY_REVIEW.md`.

## Authorized Testing

Penetration testing requires explicit authorization and scope. Use
`PENTEST_SCOPE_TEMPLATE.md` before testing.
